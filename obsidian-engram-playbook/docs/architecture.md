# Architecture

## Overview

This playbook separates the system into three layers:

1. **Obsidian base layer**
   - Markdown conventions
   - frontmatter conventions
   - Base / Canvas / CLI / web clipping support
2. **Governance layer**
   - daily journal writing
   - weekly synthesis
   - monthly / quarterly / yearly review
   - experience summarization
3. **Memory layer**
   - Engram for reusable patterns, anti-patterns, rules, and handoffs

## Design principles

### Default-on foundation
Foundation skills should feel invisible.
Users describe the task; agents attach the right base capability.

### Evidence-first outputs
High-level synthesis must be traceable to notes, links, files, or explicit user facts.

### Minimal file proliferation
The system should resist creating files just to feel structured.

### Memory is not the vault
The vault is for readable work products.
Engram is for reusable compressed memory.

## Typical flow

Daily work -> daily journal -> weekly report -> monthly review -> reusable lessons into Engram
