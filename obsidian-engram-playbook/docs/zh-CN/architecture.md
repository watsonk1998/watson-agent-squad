# 架构说明

## 总览

这个 playbook 把系统分成三层：

1. **Obsidian 基础层**
   - Markdown 规范
   - frontmatter 规范
   - Base / Canvas / CLI / 网页剪藏支持
2. **治理层**
   - 日记写作
   - 周报综合
   - 月报 / 季报 / 年报复盘
   - 经验总结
3. **记忆层**
   - Engram 用于保存可复用模式、反模式、规则和交接信息

## 设计原则

### 基础层默认生效
基础能力应该尽量“隐形”。
用户只描述任务，Agent 自动接入正确的底层能力。

### 证据优先
高层综合结论应尽量能追溯到笔记、链接、文件或用户明确提供的事实。

### 最小文件增殖
系统应尽量抵抗“为了结构化而结构化”的冲动，不要不断制造新文件。

### 记忆不等于 vault
Vault 用来承载可读的工作产物。
Engram 用来承载可复用、可压缩、跨会话延续的记忆。

## 典型流程

日常工作 -> 日记 -> 周报 -> 月报 -> 经验沉淀进入 Engram
