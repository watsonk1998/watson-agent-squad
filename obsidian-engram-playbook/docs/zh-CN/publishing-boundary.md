# 发布边界

## 可以公开的内容

适合放进公开仓库：
- 方法论
- 模板
- 脱敏后的 skill
- 通用脚本
- 中性示例
- 架构与治理说明

## 不应公开的内容

应保持私有：
- 真实日记与周/月报
- 真实 Engram 记忆
- 真实 MCP 配置
- 真实机器路径
- 带有人名、组织名、项目名的私有元数据
- 带可识别内容的截图或附件
- git 历史中残留的敏感上下文

## 核心原则

> 公开的是 **framework**，不是 **life data**。

## 推荐拆分方式

- **公开仓库**：框架 + 模板 + 脱敏示例
- **私有 vault / 私有仓库**：真实内容、真实配置、真实工作流数据
