# 脱敏清单

在公开或分享任何 Obsidian + Engram 工作流内容前，请逐项检查。

## 1. 路径与设备信息
- [ ] 替换 `/Users/<name>/...` 之类绝对路径
- [ ] 替换用户名、hostname、设备标签
- [ ] 替换会暴露机器信息的 launcher / service label

## 2. 身份与组织信息
- [ ] 去掉真实姓名（除非本来就打算公开）
- [ ] 替换雇主、客户、项目名
- [ ] 替换内部代号和私有仓库名

## 3. Vault 内容
- [ ] 不公开真实日记
- [ ] 不公开真实周报 / 月报
- [ ] 不公开包含真实参会人的会议纪要
- [ ] 清理带可识别信息的截图与附件

## 4. Engram / memory 数据
- [ ] 不公开原始 memory 导出
- [ ] 如果 `engram_topic_key` 含有真实项目名/人名，要改掉
- [ ] 清理与真实工作流直接绑定的 handoff 记录

## 5. 密钥与接口
- [ ] 去掉 API key / token
- [ ] 去掉 MCP endpoint 与认证配置
- [ ] 去掉 SSH host、内部域名、端口与服务地址

## 6. 元数据与链接
- [ ] 检查 `canonical_name`、`aliases`、`tags`、wikilinks
- [ ] 把私有目录名替换成中性占位符
- [ ] 检查附件名和嵌入文件名

## 7. 脚本与自动化
- [ ] 把硬编码路径改成环境变量或占位符
- [ ] 把组织专用命令改成通用示例
- [ ] 移除私有运维假设

## 8. Git 历史
- [ ] 检查 commit message 是否泄露敏感上下文
- [ ] 检查已删除文件是否仍留在历史中
- [ ] 若曾提交敏感信息，必要时重写历史

## 9. 最终检查
